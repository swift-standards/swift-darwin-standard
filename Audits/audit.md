### Summary

11 findings: 0 CRITICAL, 9 HIGH, 1 MEDIUM, 1 LOW.

**Systemic patterns**:

1. **Inconsistent file-level gating** — 18 of 22 Swift source files have `#if os(macOS) || ... || os(visionOS)` or `#if canImport(Darwin)` at the top; the Darwin Loader target's three core files and the Darwin Memory Standard's Statistics file lack this gating entirely. Target-level gating via Package.swift `condition: .when(platforms: [.macOS, ...])` is also absent. Either mechanism would fix the issue; the ecosystem pattern elsewhere is file-level `#if os()`. The result is that `swift-darwin-standard` is formally cross-platform-compilable but has undefined behavior on non-Darwin.

2. **Raw C pointer types as public API surface** — 5 files expose `UnsafePointer<CChar>`, `UnsafeRawBufferPointer`, or `UnsafeRawPointer` in public signatures (Attributes.Extended, File.Stats, File.Move.Extensions, Loader.Image, Loader.Symbol+Darwin). [PLAT-ARCH-005a] mandates these be wrapped in ecosystem types for public surface; raw forms may exist under `@_spi(Syscall)`. Attributes.Extended is the worst offender (no safe overload pair); the others have safe overloads and only need the unsafe form demoted to SPI.

3. **Correct patterns observed** — `Kernel.Event.Queue` wraps `kevent` internally (C type isolated to `internal typealias CKevent` and `@_silgen_name("kevent")` internal syscall bridge). `Darwin.Kernel.Thread.ID` uses `UInt32` (matches `mach_port_t` width) per [PLAT-ARCH-015]. Namespace extension pattern [PLAT-ARCH-003/004] is correctly applied via `Darwin.Kernel = Kernel_Primitives_Core.Kernel` typealias. POSIX syscalls are NOT duplicated here per [PLAT-ARCH-007] (no fork/waitpid/signal/socket wrappers observed). `Kernel.Event.Queue` uses `~Copyable` ownership + `@_spi(Syscall) public let descriptor: Kernel.Descriptor` — correct descriptor veneer pattern per [PLAT-ARCH-005].

4. **Copyright/project-name drift** — several Darwin Kernel Standard files carry "swift-darwin-primitives" / "swift-kernel" in header; headers reference prior-organisation names. Not a platform-rule finding; tracked elsewhere.
